# Simple Prompt Archive PRD

## 1. 개요

Simple Prompt Archive는 사용자가 자주 쓰는 생성형 AI 프롬프트를 콘솔에서 빠르게 등록, 검색, 분류, 재사용할 수 있게 하는 Python 기반 프롬프트 관리 도구다. 사용자는 별도 서버나 외부 의존성 없이 로컬 터미널에서 프롬프트를 관리하고, 필요할 때 JSON 또는 Markdown 파일로 데이터를 보관하거나 공유할 수 있다.

## 2. 목표

- 프롬프트를 제목, 본문, 카테고리, 즐겨찾기 상태와 함께 구조적으로 관리한다.
- 검색, 카테고리 필터, 즐겨찾기, 조회수 정렬을 통해 필요한 프롬프트를 빠르게 찾는다.
- 실행 중에는 메모리에서 데이터를 다루고, 사용자가 명시적으로 선택할 때만 파일 가져오기와 내보내기를 수행한다.
- Python 3.13 최신 패치 계열과 표준 라이브러리만으로 실행 가능한 콘솔 도구를 제공한다.

## 3. 사용자와 사용 환경

- 대상 사용자는 텍스트, 이미지, 영상, 자동화 등 여러 목적의 프롬프트를 반복해서 사용하는 개인 사용자다.
- 기본 실행 환경은 로컬 터미널이다.
- UI는 번호 입력 기반 콘솔 메뉴로 제공한다.
- 외부 패키지, 데이터베이스, 네트워크 연결 없이 동작해야 한다.

## 4. 기능 요구사항

### 4.1 메뉴와 기본 흐름

- 프로그램 시작 시 사용 가능한 기능 목록을 메뉴로 표시한다.
- 사용자는 번호를 입력해 기능을 선택한다.
- 잘못된 메뉴 번호나 처리할 수 없는 입력이 들어오면 안내 메시지를 표시하고 메뉴 흐름을 유지한다.
- 각 기능 수행이 끝나면 기본 메뉴로 돌아간다.
- 종료 메뉴를 선택하면 프로그램을 종료한다.

### 4.2 프롬프트 데이터

- 프로그램 시작 시 최소 3개의 스타터 프롬프트를 기본 데이터로 제공한다.
- 각 프롬프트는 다음 필드를 가진다.
  - `title`: 프롬프트 제목
  - `content`: 프롬프트 본문
  - `category`: 자유입력 카테고리
  - `favorite`: 즐겨찾기 여부
  - `usage_count`: 상세 조회 횟수
- 사용자가 추가한 프롬프트와 변경한 상태는 실행 중 메모리에만 유지된다.
- 프로그램은 자동 저장을 수행하지 않는다.
- 동일하거나 유사한 제목과 본문을 가진 프롬프트도 별도 항목으로 등록할 수 있다.

### 4.3 프롬프트 추가

- 사용자는 제목, 본문, 카테고리를 입력해 새 프롬프트를 추가할 수 있다.
- 제목, 본문, 카테고리 중 하나라도 비어 있으면 다시 입력을 요청한다.
- 카테고리는 고정 목록 없이 사용자가 원하는 문자열을 입력한다.
- 새 프롬프트의 기본 `favorite` 값은 `false`다.
- 새 프롬프트의 기본 `usage_count` 값은 `0`이다.
- 새 프롬프트는 목록의 마지막에 추가되며, 이 순서를 등록순으로 사용한다.

### 4.4 목록 보기

- 저장된 모든 프롬프트를 등록순으로 표시한다.
- 각 행에는 목록 번호, 카테고리, 제목, 즐겨찾기 표시를 포함한다.
- 목록 번호는 현재 표시된 목록에서 사용자가 항목을 선택할 수 있는 기준으로 사용한다.
- 프롬프트가 없으면 빈 상태 안내 메시지를 표시한다.

### 4.5 카테고리별 조회

- 현재 등록된 프롬프트에서 카테고리 목록을 추출해 표시한다.
- 카테고리 목록은 해당 카테고리가 처음 등장한 등록순으로 표시한다.
- 사용자가 카테고리를 선택하면 해당 카테고리의 프롬프트만 등록순으로 표시한다.
- 해당 카테고리에 프롬프트가 없으면 결과 없음 안내 메시지를 표시한다.

### 4.6 검색

- 사용자는 키워드를 입력해 프롬프트를 검색할 수 있다.
- 검색 대상은 제목과 본문이다.
- 검색은 부분 일치 방식으로 동작한다.
- 영문 검색은 대소문자를 구분하지 않는다.
- 검색 결과는 등록순으로 표시한다.
- 검색 결과가 없으면 결과 없음 안내 메시지를 표시한다.

### 4.7 상세 보기

- 사용자는 목록 번호를 입력해 프롬프트 상세 정보를 볼 수 있다.
- 상세 화면에는 제목, 카테고리, 즐겨찾기 여부, 조회수, 본문 전체를 표시한다.
- 상세 보기가 성공하면 해당 프롬프트의 `usage_count`를 1 증가시킨다.
- 잘못된 번호를 입력하면 안내 메시지를 표시하고 데이터를 변경하지 않는다.

### 4.8 즐겨찾기

- 사용자는 프롬프트 번호를 입력해 즐겨찾기를 켜거나 끌 수 있다.
- 즐겨찾기 상태가 켜진 프롬프트는 목록과 상세 화면에서 명확히 구분해 표시한다.
- 즐겨찾기 목록 기능은 `favorite` 값이 `true`인 프롬프트만 등록순으로 표시한다.
- 즐겨찾기된 프롬프트가 없으면 빈 상태 안내 메시지를 표시한다.

### 4.9 수정

- 사용자는 수정할 프롬프트를 번호로 선택한다.
- 수정 화면에서는 제목, 본문, 카테고리 중 변경할 필드를 선택한다.
- 선택한 필드의 새 값이 비어 있으면 다시 입력을 요청한다.
- 수정 기능은 `favorite`와 `usage_count`를 직접 변경하지 않는다.

### 4.10 삭제

- 사용자는 삭제할 프롬프트를 번호로 선택한다.
- 유효한 번호가 입력되면 해당 프롬프트를 즉시 삭제한다.
- 잘못된 번호를 입력하면 안내 메시지를 표시하고 데이터를 변경하지 않는다.
- 삭제 후 남은 프롬프트의 표시 번호는 현재 등록순에 맞게 다시 계산한다.

### 4.11 조회수 정렬 목록

- 사용자는 조회수가 높은 순서로 전체 프롬프트 목록을 볼 수 있다.
- 정렬 기준은 `usage_count` 내림차순이다.
- 조회수가 같은 프롬프트는 등록순을 유지한다.
- 각 행에는 목록 번호, 카테고리, 제목, 즐겨찾기 표시, 조회수를 포함한다.

## 5. 파일 입출력 요구사항

### 5.1 JSON 내보내기

- 사용자는 현재 프롬프트 데이터를 JSON 파일로 내보낼 수 있다.
- 기본 저장 경로는 `data/prompts.json`이다.
- `data/` 디렉토리가 없으면 자동으로 생성한다.
- 같은 경로에 파일이 이미 있으면 항상 덮어쓴다.
- JSON 최상위 구조는 다음 형태를 따른다.

```json
{
  "version": 1,
  "prompts": [
    {
      "title": "string",
      "content": "string",
      "category": "string",
      "favorite": false,
      "usage_count": 0
    }
  ]
}
```

### 5.2 JSON 가져오기

- 사용자는 `data/prompts.json` 파일에서 프롬프트 데이터를 가져올 수 있다.
- 가져오기가 성공하면 현재 실행 중인 프롬프트 목록을 파일 데이터로 전체 교체한다.
- 최상위 `version`이 없거나 `1`이 아니면 가져오기에 실패한다.
- 최상위 `prompts` 배열이 없으면 가져오기에 실패한다.
- 각 프롬프트 항목에는 `title`, `content`, `category`, `favorite`, `usage_count` 필드가 모두 있어야 한다.
- 필드 존재 여부만 검증하며, 값 타입은 파일 내용을 그대로 신뢰한다.
- 가져오기에 실패하면 현재 실행 중인 데이터는 유지하고 오류 안내 메시지를 표시한다.
- 가져온 뒤의 등록순은 `prompts` 배열 순서를 그대로 따른다.

### 5.3 Markdown 내보내기

- 사용자는 현재 프롬프트 데이터를 카테고리별 Markdown 파일로 내보낼 수 있다.
- 기본 저장 위치는 `data/` 디렉토리다.
- 내보내기 경로는 `data/{categoryOrder}-{categorySlug}/{promptOrder}-{promptSlug}.md` 형태를 따른다. 예시는 `data/0001-writing/0001-better-title.md`다.
- 순번은 다음 규칙을 따른다.
  - 카테고리 순번은 카테고리가 처음 등장한 등록순으로 부여한다.
  - 프롬프트 순번은 해당 카테고리 안에서의 등록순으로 부여한다.
  - 카테고리 순번과 프롬프트 순번은 모두 `0001`부터 시작하는 고정 4자리다.
  - 카테고리 수 또는 카테고리별 프롬프트 수가 9,999개를 초과하면 내보내기에 실패한다.
- slug는 다음 규칙을 따른다.
  - 한글, 알파벳, 숫자만 유지한다.
  - 알파벳은 소문자로 변환한다.
  - 공백과 허용되지 않는 문자의 연속 구간은 `-`로 바꾼다.
  - 앞뒤 `-`는 제거한다.
  - slug가 비어 있으면 내보내기에 실패한다.
- 파일시스템 변경은 모든 경로를 사전 검증한 뒤 수행한다.
  - 사전 검증에 실패하면 `data/`와 카테고리 디렉토리를 포함해 아무 파일시스템 변경도 하지 않는다.
  - `data/` 위치에 파일이 있거나, 필요한 폴더 위치에 파일이 있거나, 필요한 파일 위치에 폴더가 있으면 내보내기에 실패한다.
  - 사전 검증을 통과하면 이번 내보내기 대상 파일만 덮어쓴다.
  - 이번에 생성되지 않는 기존 파일이나 폴더는 삭제하지 않는다.
- 프롬프트가 0개이면 `data/` 디렉토리만 보장하고 Markdown 파일 없이 성공 처리한다.
- 각 Markdown 파일은 YAML 스타일 frontmatter와 본문으로 구성한다.
  - frontmatter 필드는 `title`, `category`, `favorite`, `usage_count`다.
  - `title`과 `category`는 큰따옴표로 감싼다.
  - 큰따옴표, 역슬래시, 제어문자는 JSON 방식으로 escape한다.
  - 한글 등 비ASCII 문자는 escape하지 않고 그대로 보존한다.
  - `favorite`은 `true` 또는 `false`로 쓰고, `usage_count`는 숫자로 쓴다.
  - frontmatter 이후 본문 전체를 `content`로 간주한다.

```md
---
title: "Better title"
category: "writing"
favorite: false
usage_count: 0
---

프롬프트 본문
```

## 6. 비기능 요구사항

- Python 3.13 최신 패치 계열에서 실행되어야 한다.
- Python 표준 라이브러리만 사용해야 한다.
- 프로그램은 콘솔에서 읽고 이해하기 쉬운 안내 문구를 제공해야 한다.
- 기능별 코드는 유지보수하기 쉽게 분리되어야 한다.
- 사용자 입력 오류가 프로그램 비정상 종료로 이어지지 않아야 한다.
- 파일 입출력 실패 시 현재 메모리 데이터가 손상되지 않아야 한다.

## 7. 수용 기준

- 프로그램 시작 직후 최소 3개의 스타터 프롬프트가 목록에 표시된다.
- 새 프롬프트를 추가하면 등록순 목록 마지막에 표시된다.
- 제목 또는 본문에 포함된 키워드로 프롬프트를 검색할 수 있다.
- 카테고리별 조회는 자유입력 카테고리를 기준으로 동작한다.
- 상세 보기를 수행하면 해당 프롬프트 조회수가 1 증가한다.
- 즐겨찾기 토글 후 즐겨찾기 목록에 반영된다.
- 제목, 본문, 카테고리 중 선택한 필드를 수정할 수 있다.
- 프롬프트 삭제 후 목록 번호가 현재 목록 기준으로 다시 표시된다.
- JSON 내보내기 후 `data/prompts.json`에 지정된 구조로 데이터가 저장된다.
- 유효한 JSON 파일을 가져오면 현재 목록이 파일 데이터로 전체 교체된다.
- 유효하지 않은 JSON 파일을 가져오면 기존 목록이 유지된다.
- Markdown 내보내기 후 `data/0001-.../0001-....md` 형태의 카테고리별 Markdown 파일이 생성된다.
